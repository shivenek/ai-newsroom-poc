Machine Payments Protocol: Stripe lässt KI-Agenten jetzt selbst bezahlen — ohne dich

Stripe und das Blockchain-Startup Tempo haben ein Open-Source-Protokoll veröffentlicht, das KI-Agenten erlaubt, eigenständig per HTTP zu bezahlen — Kreditkarte, Buy Now Pay Later oder Stablecoin, ganz ohne menschlichen Klick. Visa schreibt an den Specs mit. Die Ära der autonomen Maschinenzahlung hat gerade eine Infrastruktur bekommen.

Was MPP ist — in einem Satz

Das Machine Payments Protocol macht Bezahlen so einfach wie einen API-Call. Ein KI-Agent fordert eine Ressource an — eine API, einen Service, eine Bestellung. Der Server antwortet mit HTTP 402: "Payment Required". Der Agent autorisiert die Zahlung. Fertig. Kein Mensch im Loop, kein Checkout-Formular, kein "Bitte bestätigen Sie Ihre Kreditkarte".

Das klingt nach Science-Fiction? Am 18. März 2026 ging Tempos Blockchain-Mainnet live, und Stripe hat die Dokumentation veröffentlicht. Das passiert jetzt.

Karte, Stablecoin, BNPL — alles auf einem Rail

MPP ist bewusst zahlungsmethoden-agnostisch. Für Crypto-Zahlungen nutzt das Protokoll On-Chain-Deposits über das Tempo-Netzwerk. Für klassische Zahlungen — Kreditkarten, Wallets, Buy Now Pay Later — gibt es sogenannte Shared Payment Tokens (SPTs), die über Stripes bestehende Rails abgewickelt werden. Für Businesses sieht das aus wie eine normale Stripe-Transaktion: Betrugsschutz, Steuerberechnung, Auszahlung in der Standardwährung. Kein neues Backend nötig.

Die ersten Use Cases laufen bereits: Browserbase lässt Agenten pro Headless-Browser-Session bezahlen. PostalForm druckt und verschickt Briefe auf Agent-Befehl. Prospect Butcher Co. — ja, ein Metzger — liefert Essen aus, bestellt von einer KI. Stripe Climate nimmt programmgesteuerte Spenden entgegen.

Stripe, Tempo, Visa — das Gewicht hinter dem Standard

Tempo ist kein Garagen-Startup. Backed von Stripe selbst und Paradigm, bewertet mit fünf Milliarden Dollar. Visa hat die Spezifikationen für Kartenzahlungen beigesteuert. Das Protokoll ist Open Source und erweiterbar auf andere Blockchains. Wenn du dich fragst, ob das ernst gemeint ist: Ja. Die Infrastruktur-Schwergewichte sitzen am Tisch.

Aber sie sitzen nicht allein. Coinbase hat mit x402 einen eigenen Standard veröffentlicht, der denselben HTTP-402-Mechanismus nutzt, aber stärker auf Crypto fokussiert. Visa treibt parallel eine Bridge-Partnerschaft für Stablecoin-Karten-Issuing voran — das klassische Kartennetzwerk will den Anschluss an die Agent Economy nicht verlieren. Was wir hier sehen, ist kein einzelnes Produkt — es ist ein Standardkampf um die Zahlungsinfrastruktur der Agent Economy.

Was das für dich bedeutet — wenn du was verkaufst

Wenn dein Produkt eine API hat, einen SaaS-Service, irgendetwas, das ein Agent konsumieren könnte: MPP-Kompatibilität ist ab jetzt eine Produktanforderung. Nicht in zwei Jahren, jetzt. Die Stripe-Docs stehen online, die Integration läuft über die bekannte PaymentIntents-API.

Der Grund ist simpel: Agentic Commerce wird nicht über Checkout-Seiten laufen. Agenten haben keine Browser. Sie haben HTTP-Requests. Und wer keinen 402-Endpunkt bietet, existiert für sie nicht. Das ist wie eine Website ohne Mobile-Version im Jahr 2015 — technisch möglich, strategisch tödlich.

Business Punk Check

MPP macht Bezahlen zu einem Protokoll, nicht zu einem Prozess. Das ist der Moment, in dem Maschinen von Nutzern zu Kunden werden. Stripe, Visa und fünf Milliarden Dollar Tempo-Bewertung sorgen dafür, dass das nicht im Whitepaper stecken bleibt.

Wenn du SaaS, APIs oder Services verkaufst und keinen Plan für maschinelle Zahlungseingänge hast, bist du in zwei Jahren der Laden ohne Kartenlesegerät. Der Standardkampf zwischen MPP und Coinbases x402 entscheidet, wer die Rails besitzt. Positionier dich — oder werde positioniert.

Häufig gestellte Fragen

Was genau ist das Machine Payments Protocol?

Ein Open-Source-Standard von Stripe und Tempo, der KI-Agenten erlaubt, per HTTP-Request eigenständig Zahlungen auszulösen — ohne menschliche Freigabe.

Welche Zahlungsmethoden unterstützt MPP?

Kreditkarten, Wallets, Buy Now Pay Later (über Stripes Rails) und Stablecoins (über das Tempo-Blockchain-Netzwerk). Erweiterbar auf andere Blockchains.

Was ist der Unterschied zwischen MPP und Coinbases x402?

Beide nutzen den HTTP-402-Mechanismus. MPP ist zahlungsmethoden-agnostisch (Fiat und Crypto), x402 fokussiert stärker auf Crypto-native Zahlungen. Es ist ein Standardkampf um die Infrastruktur der Agent Economy.

Muss ich als Unternehmen jetzt sofort handeln?

Wenn dein Produkt von KI-Agenten konsumiert werden könnte — ja. Stripe bietet bereits Dokumentation und Early Access für MPP-Integration. Wer wartet, wird für Agenten schlicht unsichtbar.

Wie wird Betrug bei autonomen Agentenzahlungen verhindert?

MPP-Zahlungen über Stripes Fiat-Rails nutzen dieselben Fraud-Protection-Mechanismen wie normale Stripe-Transaktionen — inklusive Betrugserkennung, Spending Limits und Steuerberechnung. Agenten operieren innerhalb vordefinierter Autorisierungsgrenzen, die der Auftraggeber festlegt. Kein Agent hat Blankovollmacht.
